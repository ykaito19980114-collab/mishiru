## 1. 背景と課題定義（00の確定版要約）

学生は情報不足ではなく**検索語・判断軸・自己理解の不足**で研究室探索を開始できない。教授は**宣伝への抵抗と編集負荷**で価値ある情報の発信を止めている。既存の研究室検索・口コミポータル（LabBase等）と正面勝負せず、「検索前の興味関心探索」×「研究室情報整備の業務代行」の空白を取る（00 §1-2参照）。アンチゴール：ランキング・無断口コミ・評価サイト・学生情報販売は**やらない**。

## 2. プロダクトビジョンとコンセプト

### 2.1 一文定義（PV-01 / Must）

> **検索語を持たない学生の興味関心を研究テーマ・研究室へ接続し、研究室側の情報発信を業務として支える B2B2C サービス。**

合格ライン：一文説明が「研究室検索サービス」**だけ**に見えないこと。

### 2.2 核となる変換

- これまで「研究室名・分野名を知っている学生しかできなかった」研究室探しを、**誰でも日常の興味から始められる**ようにする。
- これまで「教授が自力で書くか、放置するか」だった研究室ページを、**診断→制作→運用の業務代行で維持される資産**に変える。

### 2.3 ポジショニング宣言（POS-01 / Must）

本サービスは「興味関心の探索」と「研究室情報の整備」のサービスであり、以下では**ない**：

- 研究室ランキングサイトではない。教授評価サイトではない。
- 無断口コミ・暴露プラットフォームではない。
- 論文推薦エンジン単体ではない。
- 全国研究室検索DB・企業向け研究室DB単体ではない（LabBase等の土俵に乗らない）。

合格ライン：営業資料・LP・アプリ内文言に禁止表現（Tinder/ランキング/口コミ/マッチングアプリ的表現）がないこと（Test-PROH-01）。

### 2.4 コピー方針

- **禁止する見せ方**：「研究室版Tinder」「スワイプで運命の研究室」「教授の評判がわかる」等の軽薄・評価的表現。詩的・抽象的コピー。断定的な診断表現（「あなたは◯◯タイプです」）。
- **標準の見せ方**：学生向け＝専門知識なしで読める平易さ（例:「興味から、研究テーマと研究室が見つかる」）。教授向け＝軽く見せない正確性（「学生への情報提供」「ミスマッチ防止」「研究成果発信」）。専攻向け＝業務支援としての堅さ。診断は常に「傾向」と表現。

## 3. ターゲットユーザーとゴール

- **プライマリペルソナ**：P1 佐々木航（検索語を持たない院進検討学生）／顧客側 P3 高梨真（新任・多忙PI）。**セカンダリ**：P2 李美怜（直前まで動けない非消費層）、専攻・研究科、学生ライター。（詳細は00 §3.1）
- **ユーザーゴール**：学生＝自分の興味の輪郭を知り、納得感を持って候補研究室に進む。教授＝手を動かさずに、正確で学生に伝わる研究室ページを持つ。専攻＝研究室群を比較可能な形で見せ、院試・配属導線を強くする。
- **プロダクトゴール**：学生の利用実績（保存・遷移）を証拠として、教授/専攻からの制作・運用収益を成立させる。
- **UX構成式によるゴール分解**：
  - u最大化：初回セッションで「興味プロファイル＋候補研究室」という具体的な持ち帰りを渡す。教授には「診断レポート」という具体的な改善提案を渡す。
  - f最小化：**初回3分以内・5タップ以内**でコア価値到達（BR-01）。入力を求めない（匿名で開始）。専門用語を前提にしない。
  - e最大化：知的関心が整理される納得感。押し付けない語り口（傾向表現）。研究の面白さが伝わる編集品質。

## 4. ステークホルダー分析

### 4.1 影響力×利害の4象限

| | 利害:大 | 利害:小 |
|---|---|---|
| **影響力:大** | 教授/PI（顧客・情報主体）、専攻/研究科（束売り顧客・決裁者） | 大学広報・情報システム部門（掲載への許容/警戒） |
| **影響力:小** | 学生（利用者）、学生ライター（供給者） | 保護者、企業（将来顧客） |

### 4.2 反対者・懐疑的ステークホルダーと対処方針

| 反対者 | 反対理由 | 対処 |
|---|---|---|
| 掲載される教授（未接触段階） | 無断掲載感・誤情報への不安・「軽いサービス」への嫌悪 | 公開情報のみ＋出典明示＋未確認表示＋1営業日以内の修正/非公開SLA（STR-PI-02）。営業文言規範（§2.4） |
| 大学広報・法務 | 商標・著作権・学生への影響懸念 | 本文転載なし（K-LAW-03）・独立第三者サービスの明示・削除窓口常設 |
| 競合（LabBase等） | 市場重複 | 土俵をずらす（BR-04）。外部説明でも比較語を使わない |
| 学生ライターの所属研究室 | 内部情報の漏洩懸念 | 自由投稿なし。教授確認フロー必須（FR-ARTICLE-01） |

## 7. ビジネスモデル

- **収益モデル**：①見え方診断（無料〜低額・営業導線）→ ②研究室ページ制作 15〜30万円 → ③月額運用 3万円〜（記事更新・成果更新・学生テーマ追加）→ ④専攻束売り 30〜100万円 → ⑤（将来）企業向け調査レポート。学生は恒久無料。
- **コスト構造・ユニットエコノミクス**：00 §7参照。制作20h・運用3h/月が損益分岐の運用条件（BR-06）。
- **成長モデル**：獲得＝教授の身内紹介→専攻内紹介（束売り）→学会・大学広報経由。学生側は院試・配属シーズンのSNS/検索流入。**ネットワーク効果**：公認研究室ページが増える→学生の候補提示品質が上がる→利用実績が営業証拠になる、の循環。

## 9. 成功指標（KGI/KPI）とGo/No-Go基準

**KGI**：M1終了時点で有償制作3件・うち月額運用1件以上（＝「払われる事業である」ことの証明）。

| KPI | 合格ライン | 未達時の判定 | 測定対象 |
|---|---|---|---|
| KPI-01 カード保存率 | 60%以上 | カード粒度・文言・カテゴリを修正（開発を進めない） | 学生15〜30人 |
| KPI-02 診断完了率 | 50%以上 | 質問数・初回導線を削る | 学生15〜30人 |
| KPI-03 研究室ページ遷移率 | 30%以上 | 接続理由が弱い。論文でなく学生向け説明に変換し直す | 学生15〜30人 |
| KPI-04 教授面談/有償 | 面談3/5・有償or紹介1/5 | 営業対象・文言・価格を見直す | 教授5名 |
| KPI-05 制作工数 | 20時間以内/件・運用3時間以内/月 | テンプレートとスコープを削る | 制作3件 |
| KPI-06 専攻有用性 | 有用2/3・予算ルート1/3 | 専攻提案を後ろ倒しし教授個別営業を継続 | 専攻関係者3名 |

**撤退基準**：00 §8のとおり（学生3指標全滅→カード仮説棄却しB2B単体へ縮小／教授有償0×2巡→専攻束売りへピボット）。判断を機能追加で先送りしない。
