# ADR-007: 全画面ビジュアルリデザイン「Clear Board, One Blue」

- 日付: 2026-07-17
- 状態: 採用
- 起点: ユーザー要望「フォントサイズや余白が全体的に見にくい。サウナイキタイ(https://sauna-ikitai.com/)を参考に、洗練されていながらキャッチーに。要素を削ぎ落とし、初見でも迷いなく使える、視認性・整理性の高いUIUXへ」

## CONCEPT宣言

```
CONCEPT: Clear Board, One Blue（白い掲示板と、一筆の青）
構図: バンド構成（青のヒーロー帯＝記憶の1点）＋圧縮リスト行
タイポ表現: ①ジャンプ率の回復（見出しw900と本文w400の明確な対比）②数字がヒーロー（tabular-nums・青い大数字）
モーション個性: 編集的（2〜6px・150ms・reduced-motionで全停止）
質感: フラット白カード＋1px罫線（グラデ・ノイズ・大影を全廃）
記憶に残る1点: さがすヒーローの「青いベタ帯」＝ブランド青の面に白い検索箱ひとつ
定型からの逸脱: 一覧から文章を追放（Q.は1行だけ）／「白カード+青ラベル」等リズムの解体／FABの主張降格
```

## サウナイキタイから移植する7原則（本リデザインの判断基準）

1. **1画面=1主アクション**。競合するCTAを同格で並べない。第一アクション以外は視覚的に一段落とす。
2. **色は「地の無彩色＋二色の役割分担」**（2026-07-17改訂）。**青＝行動**（進む・リンク・主ボタン・情報の強調）、**ライム＝選択と現在地**（アクティブなタブ/ナビ・チェック済み・テキスト選択・メモFAB・検索ボタン）。ライムは面と細部にのみ使い**文字色には使わない**（ライム面上の文字は常にink、コントラスト12:1）。全体面積の5〜15%に留めて希少価値を守る。装飾目的の色は引き続き禁止。
3. **数字がヒーロー**。件数・カウント・温度計的な事実は大きく太く（tabular-nums）。文章は小さく細く。
4. **一覧に文章を置かない**。リスト行は 名詞＋数字＋チップ＋(最大)1行フック で構成。段落は詳細ページの下層のみ。
5. **チップは灰1種＋青1種のみ**。灰=属性、青(淡青地)=分野/選択。意味のない色分けをしない。
6. **段階的開示**。説明・免責・補足は12px灰色1行に圧縮し、詳細はdetails/展開に隠す。UIが自分の仕組みを語らない（「何をしないか」を先に言わない）。
7. **操作は端に固定**。詳細ページの主アクションは下部固定バー(最大2つ+外部リンク)。一覧の行は行全体がリンク（行内にボタンを置かない）。

## デザイントークン v2（index.cssの:rootを唯一の出所とする）

```css
/* 色 */
--c-bg: #f4f4f2;            /* 地。グラデ・ドットノイズは全廃 */
--c-paper: #ffffff;         /* カード面 */
--c-surface: #ececea;       /* 窪み面（セグメント地・入力地） */
--c-ink: #17181c;           /* 見出し・本文強 */
--c-ink-2: #45474f;         /* 本文 */
--c-ink-3: #7b7e87;         /* メタ・免責・placeholder */
--c-primary: #1e3ff0;       /* 唯一の青（旧#123ef5/#1e40ffを統一） */
--c-primary-strong: #1330c4;
--c-primary-soft: #eef1ff;  /* 青の淡面: 分野チップ・Q記号地・情報ハイライト */
--c-signal: #dce51b;        /* ブランドライム＝選択・現在地（2026-07-17ユーザー指示で復帰） */
--c-signal-strong: #c6d900;
--c-signal-soft: #f5f8cf;
--c-danger: #d63a43; --c-success: #2b8a57;
--c-border: #e3e3e0;        /* 罫線は淡く */
--c-border-strong: #c9c9c5;

/* 形 */
--radius-card: 14px; --radius-panel: 10px; --radius-btn: 10px;
--shadow-sm: 0 1px 2px rgba(23,24,28,.05);          /* ほぼフラット */
--shadow-float: 0 10px 30px rgba(23,24,28,.14);     /* FAB/シート等の浮遊物のみ */

/* タイポスケール（Noto Sans JP） */
display: clamp(24px,4.5vw,34px)/w900/lh1.25   … ヒーロー・ページ題のみ
h1: 21px/w900/lh1.35    h2: 16.5px/w800/lh1.5（セクション見出し）
body: 15px/w400/lh1.85  body-strong: 15px/w700
meta: 12.5px/w500/lh1.6（灰色。大学・専攻・免責・補足）
num-hero: 26〜30px/w900/tabular-nums（統計・件数）
label: 11px/w800/letter-spacing .12em（英字eyebrow） 
```

**タイポ規律**: 本文にw700以上を使わない（強調はw700を語単位まで）。見出しw800未満禁止。フォントサイズは上記スケール外を新設しない。数字は全て `font-variant-numeric: tabular-nums`。

**余白規律**: 8pxの倍数のみ。カード内padding 16〜20px。セクション間は32〜40px（見出しの上>下）。1画面(モバイル)で研究室行が3件以上見えること。

## 共通コンポーネント規律

- **Button**: primary=青ベタ白字 / secondary=白地+ink罫線 / ghost。**灰色ベタの主ボタン禁止**（無効はopacity.4で表現し、入力が満ちたら即primary色に）。min-h 48px。
- **Chip**: 灰(bg--c-surface/文字ink-2)・青(bg--c-primary-soft/文字--c-primary)・ライム薄(bg--c-signal-soft/文字ink=保存済み等の状態)の3種。teal廃止。12.5px/w600。
- **LabRow（一覧の研究室行・最重要部品）**: 行全体が1リンク。構成は上から
  `大学・専攻・県(meta 1行)` → `研究室名(16.5px w800 1行省略)` → `PI名+職位・教員n名(meta)` → `Q. 問い1行(15px、Q.記号のみ青w900、本文ink-2、1行clamp)` → `分野チップ(青)+KWチップ(灰,最大2)`。
  行高目安128〜148px。行内にボタン・ハート・チブロン円を置かない。押下=詳細へ。
- **StatTile**: ラベル(meta)+数字(num-hero)。アイコンは16px線画1つまで。
- **免責/AI注記**: 12.5px ink-3の1行+ⓘ。長文はdetailsに畳む。本文サイズで免責を書かない。
- **セグメント/タブ**: 高さ44px。選択=primary青ベタ白字 or ink下線2px、非選択=ink-2。件数バッジは灰。
- **入力**: 白地・1px border(--c-border-strong)・focus時にborder--c-primary+3px淡青リング。高さ48〜56px。
- **FloatingMemoButton**: ライム地+ink文字の小型ピル（2026-07-17復帰）。モバイルはアイコンのみに縮む。z優先度は主CTAより下。
- **スケルトン/ロード**: 実レイアウトの骨格を灰で描く（巨大な無地ボイド禁止）。AI生成待ちは「何をしているか+目安秒数」を1行添える。
- **モーション**: hover/pressは色と1pxの沈み込みのみ。出現は2〜6px+opacity 150ms。`prefers-reduced-motion`で全停止。

## 語彙の平易化（表示コピーのみ。API・データキー・ルートは不変）

| 現行UI語 | 新表示 |
|---|---|
| 素材/材料 | 気になったこと |
| 本棚 | 保存した問い |
| 骨子 | 研究プラン |
| マーキング | メモ |
| 関心の現在地を更新 | いまの関心を分析する |
| 説明文の否定形（「〜はここでは作りません」「自動分析は行いません」） | できることだけを1行で言う |

各タブ冒頭の説明は**最大1行**。ページ見出し直下に段落を積まない。

## ページ別仕様（要点）

- **Labs(さがす)＝見本ページ**: ①ヒーロー=青ベタ帯(角丸なしフルブリード)。中は 見出し(白w900)+白い検索箱(56px,内蔵虫眼鏡)+ライム「さがす」ボタン+例チップ(白ゴースト)+もっと見る（サブ説明文は置かない=2026-07-17ユーザー指示で削除）。装飾3D画像・クリーム面・ラベンダー面・二重見出しを廃止。②帯直下: エンティティ切替(研究室/研究領域/学会/ジャーナル)=白カードのセグメント+件数(19,785は数字ヒーロー)。③絞り込み(大学から探す等)は1行のフィルタバーに集約。④結果=LabRow連続(区切り線)。
- **LabDetail**: ヘッダー(戻る/大学meta/名前display/PI/チップ)→**下部固定アクションバー**(♥気になる=secondary、保存=primary、公式サイト=ghost外部)に集約し、本文中の巨大な公式サイト枠を撤去。セクションは h2+本文15px。学生ガイドの4項目は小見出し+短文。**免責の「参考情報」縦書きレイアウト崩れを修正**し12.5px1行+detailsへ。論文は番号付きリスト(タイトル/年・被引用は数字チップ)。
- **Discover(であう)**: 説明1行。デッキカード1枚に視線集中(タイトル/フック1行/Q最大2/面白さ2行clamp)。下部固定3ボタン(違うかも=ghost/保存=secondary/気になる=primary)。「あと10枚」=細プログレスバー。ロード=カード骨格スケルトン+「AIがカードを準備しています(目安10秒)」。
- **Saved(ためる)**: 空状態は現行踏襲(コピーだけ平易化)。一覧はLabRow+メモ行。
- **Reflect(みつめる)**: 統計4タイルをStatTile化。説明の否定文を削除。主ボタンをprimary化(残回数はmeta)。「マーキング」等の未導入語を使わない。
- **Questions(研究にしてみる)**: 見出し→すぐ入力(textarea)へ。「本棚を見る」は右上ghost。ステッパーは小型の1/4表記+現ステップ名のみ。ラベルを語彙表で置換。
- **Projects/ProjectDetail/Consult**: 同トークン適用・見出し階層・チップ統一・段落を短く。
- **Universities/UniversityDetail/Departments/DepartmentDetail/CardDetail/Saved**: 一覧をLabRow様式(名詞+数字+チップ)に圧縮。
- **ForLabs/Claim/Policy/AuthCallback/admin**: トークン・タイポ置換のみ(構成不変)。
- **Layout**: bodyのグラデ/ドットノイズ/クリーム帯を撤去しフラット--c-bg。ヘッダー=白+1px下罫線(青上罫線廃止)。サイドバー(md+)のコピーカードからライムグラデ撤去。下部タブ現行4つ維持(アクティブ=inkラベル+ライムマーカー)。site-footerのライム上罫線→--c-border。
- **選択状態の統一（2026-07-17）**: サイドナビのアクティブ・検索対象セグメントのアクティブ・チェック済みフィルタ・::selection・ヒーロー検索ボタン=ライム地+ink文字。行動ボタン(primary)と情報ハイライトは青のまま。

## 不変条件（違反したら失格）

- サーバー/API/ルート/状態管理/機能の追加削除をしない。表示層(tsx/css)とコピーのみ。
- 既存のaria属性・role・focus管理・skip-link・キーボードトラップ対応を維持または改善。
- コントラスト4.5:1以上(全テキスト)。タップターゲット44px以上。h1はページ1個。
- PROH: ランキング/口コミ/Tinder的表現を導入しない。「未確認」表示・AI推定明示は必ず残す(小さくしてよいが消さない)。
- 事実・件数の捏造禁止。placeholder文言は実データに即す。
- 全色・角丸・影・フォントサイズはトークン経由。ハードコード色の新設禁止。
