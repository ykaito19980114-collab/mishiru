# ADR-001: 技術スタック — Next.js移行ではなくプロトタイプ（Vite+React+Express）を継続する

- 日付: 2026-07-03 ／ 状態: 採択
- 関連: docs/03 §1、NA-04、Excel叩き台 SPEC-STACK-01

## 文脈
Excel叩き台は「Next.js + Vercel + Supabase」を初期方針としたが、既存プロトタイプ（remix_-remix_openlab）はVite + React 19 + Express + Supabaseで構築済み。ユーザー指示は「プロトタイプをベースに実装」。

## 決定
v0はVite + React 19 + Express + TypeScriptを継続する。SSRは行わない。

## 理由
1. プロトタイプ資産（ルーティング・Supabase接続・ビルド構成）の放棄コスト > SSRのMVP便益。
2. 要件（02）にSSRを要求するものはない。NFR-PF-01はスケルトンUI＋軽量バンドルで充足可能。
3. ExpressサーバーはAPIキー秘匿・キャッシュ・冪等性（NFR-EXT-01/02）の実装点としてそのまま必要。
4. ガイドライン4-7「スタックの初手固定」：以後のセッションはこの決定に従う。

## 再評価トリガー
M2で検索流入KPIを設定し未達の場合、または研究室ページのSEO要求が事業上Mustに昇格した場合、SSR/SSG（Next.js等）を再評価する。

## 影響
- SEOはsitemap.xml・meta・構造化データ（JSON-LD）で対応（プロトタイプ実装を継承）。
- Vercelデプロイは`dist/server.cjs`のNodeランタイムで可能。
